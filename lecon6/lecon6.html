Leçon 6 : Graphes

Section 01

Journal d'apprentissage réflexif

Cette semaine, j'ai étudié les graphes, une structure de données non linéaire utilisée pour représenter des relations entre objets. J'ai compris que contrairement aux arbres, les graphes permettent des relations plus complexes : cycles, connexions multiples, pondérations.

J'ai découvert les notions de sommet, arête, arc, chemin, circuit et les différentes formes de graphes (orienté, non orienté, pondéré). Au début, j'avais des difficultés à distinguer les parcours BFS et DFS. Les exemples d'algorithmes comme Dijkstra et Prim m'ont montré l'utilité concrète des graphes dans les réseaux, GPS et bases de données.

Section 02

Synthèse des concepts clés

2.1 Définitions
Définition : Graphe

Un graphe est défini par G = (N, L) où N est l'ensemble des sommets et L l'ensemble des arcs ou arêtes.

Types de graphes :

  • Graphe orienté : les relations ont une direction (arc A → B)
  • Graphe non orienté : les relations vont dans les deux sens (arête A — B)
  • Graphe pondéré : chaque arête possède un poids (distance, coût, temps)
2.2 Concepts fondamentaux
  • Origine et extrémité : pour un arc (x, y), x = origine, y = extrémité
  • Boucle : arc (x, x) reliant un sommet à lui-même
  • Chemin : suite d'arcs reliant des sommets
  • Circuit : chemin qui revient au sommet de départ
  • Chaîne : chemin dans un graphe non orienté
  • Cycle : chaîne fermée
Graphe connexe

Tous les sommets peuvent être reliés par un chemin.

Graphe fortement connexe

Dans un graphe orienté, chaque sommet peut atteindre tous les autres.

2.3 Représentation des graphes
Liste d'adjacence

Chaque sommet contient la liste de ses voisins (tableau de listes chaînées).

1 → 2 → 4
2 → 5 → 6
3 → 6

Avantage : économise la mémoire, efficace pour graphes peu denses.

Matrice d'adjacence

Tableau à deux dimensions. 1 = arête existe, 0 = pas d'arête.

      1  2  3  4
1    0  1  0  1
2    0  0  0  1
3    0  0  0  1
4    0  0  0  0

Avantage : vérification rapide d'arête. Inconvénient : consomme beaucoup de mémoire.

2.4 Parcours de graphe
BFS : Parcours en largeur (Breadth First Search)

Visiter les voisins d'abord, en utilisant une file FIFO.

Complexité : O(|N| + |L|)

Applications : GPS, réseaux, diffusion de messages, chemin minimum (non pondéré).

DFS : Parcours en profondeur (Depth First Search)

Aller le plus loin possible, en utilisant une pile LIFO.

Complexité : O(|N| + |L|)

Applications : détection de cycles, tri topologique, composantes fortement connexes.

2.5 Plus court chemin
Algorithme de Dijkstra

Trouve le plus court chemin depuis un sommet source dans un graphe pondéré.

Condition : poids positifs.

Principe :

  • Choisir le sommet avec la plus petite distance
  • Mettre à jour les distances des voisins
  • Répéter jusqu'à tout visiter
2.6 Arbre couvrant minimal (MST)
Arbre couvrant minimal

Un arbre couvrant contient tous les sommets sans cycle. L'arbre couvrant minimal a la somme des poids minimale.

Algorithme de Prim

Principe :

  • Commencer par un sommet
  • Choisir l'arête la moins coûteuse
  • Ajouter le sommet
  • Répéter jusqu'à couvrir tout le graphe

Autre algorithme similaire : Kruskal.

Section 03

Application pratique dans mon contexte

Les graphes sont omniprésents dans le monde réel :

  • Les réseaux sociaux modélisent les utilisateurs comme sommets et les relations d'amitié comme arêtes.
  • Les GPS et systèmes de navigation utilisent Dijkstra pour trouver le chemin le plus court entre deux points.
  • Les réseaux informatiques utilisent les arbres couvrants minimaux (Prim/Kruskal) pour optimiser le câblage.
  • Les systèmes de recommandation exploitent les graphes pour suggérer du contenu basé sur les connexions entre utilisateurs.
