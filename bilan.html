Leçon 1

Bilan : Preuve et Complexité Algorithmique

Ce que j'ai le plus appris :

Cette leçon m'a fait comprendre que tester un algorithme ne suffit pas à prouver qu'il est correct. La distinction entre terminaison et correction partielle m'a montré qu'il faut raisonner mathématiquement pour garantir le bon fonctionnement d'un algorithme. J'ai appris à utiliser les invariants de boucle pour prouver formellement la correction d'un algorithme itératif, et les variants pour garantir sa terminaison. J'ai aussi approfondi la notation Big-O avec sa définition formelle et l'analyse d'algorithmes comme PrefMoyen.

Compétences réutilisables :

  • Prouver la correction d'un algorithme : construire un invariant de boucle et vérifier initialisation, conservation et conclusion.
  • Analyser la complexité : évaluer le nombre d'opérations d'un algorithme avec la notation Big-O.
  • Optimiser le code : remplacer un algorithme O(n²) par un O(n log n) dans mes projets.
  • Raisonner sur les limites du calcul : comprendre pourquoi certains problèmes (NP-complets) n'ont pas de solution efficace connue.

Mon plus grand défi :

La construction d'invariants de boucle pour des algorithmes complexes. Le passage du raisonnement intuitif au raisonnement formel reste exigeant.

Prochaines étapes :

  • Pratiquer les preuves formelles : résoudre au moins 10 exercices de construction d'invariants sur des algorithmes variés.
  • Maîtriser les récurrences : apprendre à résoudre les équations de récurrence avec le théorème maître.
  • Implémenter et analyser : coder 3 algorithmes récursifs, puis analyser formellement leur complexité.
Leçon 2

Bilan : Analyse Asymptotique des Algorithmes

Ce que j'ai le plus appris :

Cette leçon m'a appris à analyser formellement l'efficacité d'un algorithme grâce aux notations asymptotiques (Big-O, Oméga, Theta). J'ai compris que l'objectif n'est pas de mesurer un temps exact, mais d'étudier la tendance de croissance. J'ai aussi étudié les algorithmes de tri classiques (insertion, sélection, bulles), leurs pseudo-codes et leurs complexités.

Compétences réutilisables :

  • Identifier le terme dominant : simplifier une expression de complexité pour obtenir la classe asymptotique.
  • Choisir le bon algorithme de tri : comparer insertion sort, selection sort et bubble sort selon le contexte.
  • Anticiper les performances : utiliser l'analyse asymptotique pour prédire le comportement d'un programme sur de grandes données.
  • Optimiser avec les structures de données : remplacer une recherche linéaire par une table de hachage ou un arbre équilibré.

Mon plus grand défi :

L'analyse de fonctions complexes pour identifier rapidement le terme dominant. Les preuves formelles des bornes asymptotiques (trouver les constantes c et n₀) restent aussi un exercice qui demande de la pratique.

Prochaines étapes :

  • Étudier les tris avancés : Merge Sort, Quick Sort, Heap Sort et comprendre pourquoi ils atteignent O(n log n).
  • Résoudre des exercices d'analyse : pratiquer l'identification du terme dominant sur des expressions variées.
  • Comparer les implémentations : coder les 3 algorithmes de tri et mesurer leurs performances sur différentes tailles de données.
Leçon 3

Bilan : Structures algorithmiques fondamentales

Ce que j'ai le plus appris :

J'ai compris que tout algorithme repose sur trois structures fondamentales : séquentielle, conditionnelle et répétitive. Les boucles POUR et TANT QUE permettent d'automatiser le traitement de données.

Compétences réutilisables :

  • Organiser la logique d'un programme : structurer le flux d'exécution avec les bonnes structures.
  • Utiliser les boucles : automatiser les tâches répétitives et parcourir des collections de données.
  • Prendre des décisions : utiliser les conditions pour adapter le comportement d'un algorithme.

Mon plus grand défi :

Choisir la structure la plus adaptée au problème donné et optimiser l'utilisation des boucles pour l'efficacité.

Prochaines étapes :

  • Pratiquer davantage d'exercices d'algorithmique variés.
  • Analyser différents algorithmes pour identifier les structures utilisées.
  • Implémenter dans un langage de programmation pour consolider la compréhension.
Leçon 4

Bilan : Structures de données hiérarchiques

Ce que j'ai le plus appris :

J'ai découvert les arbres binaires, les ABR et les tas. Contrairement aux structures linéaires, les arbres organisent les données hiérarchiquement, ce qui améliore les performances de recherche. Les parcours préfixé, infixé et postfixé permettent de visiter tous les nœuds.

Compétences réutilisables :

  • Utiliser les ABR : recherche, insertion et suppression efficaces.
  • Appliquer les parcours d'arbre : préfixé, infixé, postfixé via la récursivité.
  • Gérer les files de priorité avec les tas (max-heap, min-heap).

Mon plus grand défi :

Les algorithmes de parcours et les opérations d'insertion/suppression dans un ABR restent à consolider par la pratique.

Prochaines étapes :

  • Pratiquer les exercices sur les arbres binaires.
  • Implémenter les structures dans un langage de programmation.
  • Analyser la complexité des opérations sur les arbres.
