Leçon 2 : Analyse Asymptotique des Algorithmes

Section 01

Journal d'apprentissage réflexif

Dans cette leçon, j'ai approfondi la manière d'analyser l'efficacité des algorithmes à l'aide des notations asymptotiques. J'ai compris que l'objectif n'est pas de mesurer un temps exact d'exécution, mais d'étudier la croissance du temps d'exécution lorsque la taille des données augmente.

La notation Big-O m'a permis de formaliser cette idée en ne conservant que le terme dominant d'une fonction de coût. Au début, il n'était pas évident pour moi de comprendre pourquoi on ignore les constantes et les termes de plus faible ordre. Cependant, les exemples comparant différentes fonctions de complexité m'ont aidé à comprendre que, pour de grandes valeurs de n, ces éléments deviennent négligeables.

Mon principal défi reste l'identification rapide du terme dominant dans certaines expressions complexes.

Section 02

Synthèse des concepts clés

2.1 Objectif de l'analyse asymptotique

L'analyse asymptotique permet d'étudier le comportement d'un algorithme lorsque la taille de l'entrée devient très grande. On ne cherche pas un temps exact mais une tendance de croissance. On modélise le temps d'exécution par :

T(n)

où n représente la taille de l'entrée.

2.2 Notation Big-O

La notation Big-O donne une borne supérieure asymptotique du temps d'exécution.

Définition : Notation Big-O

On dit que :

T(n) = O(f(n))

si :

∃ c > 0, ∃ n₀, ∀ n ≥ n₀ : T(n) ≤ c · f(n)

Cela signifie que la fonction f(n) majorera la croissance de T(n).

Exemple
T(n) = 4n² + 3n + 10
Pour les grandes valeurs de n, le terme dominant est n².
Donc : T(n) = O(n²)
2.3 Règles simplifiées d'analyse

Lorsqu'on analyse une fonction :

  • On garde le terme dominant
  • On ignore les constantes
  • On ignore les termes de plus faible ordre
Exemple
T(n) = 5n³ + 2n² + 7n + 4
Résultat : T(n) = O(n³)
2.4 Autres notations asymptotiques
Notation Ω (Oméga) : Borne inférieure
T(n) = Ω(f(n))

Signifie que l'algorithme prend au moins ce temps.

Notation Θ (Theta) : Borne exacte
T(n) = Θ(f(n))

Signifie que la fonction est encadrée :

c₁ · f(n) ≤ T(n) ≤ c₂ · f(n)
2.5 Classes classiques de complexité
Complexité Interprétation
O(1) Temps constant
O(log n) Logarithmique
O(n) Linéaire
O(n log n) Quasi-linéaire
O(n²) Quadratique
O(2ⁿ) Exponentielle
Section 03

Analyse d'algorithmes types

3.1 Boucle double indépendante
Pour i = 1 à n
    Pour j = 1 à n
        opération constante

Nombre d'opérations :

n × n = n²

Donc : T(n) = O(n²)

3.2 Boucle imbriquée dépendante
Pour i = 1 à n
    Pour j = 1 à i
        opération constante

Nombre total d'opérations :

1 + 2 + 3 + ... + n = n(n + 1) / 2

Donc : T(n) = O(n²)

Section 04

Algorithmes de tri

4.1 Propriétés des algorithmes de tri
In-place

Un algorithme est in-place s'il utilise très peu de mémoire supplémentaire et modifie directement le tableau original.

Exemples : tri par insertion, tri par sélection, tri à bulles.

Stabilité

Un algorithme de tri est stable si les éléments égaux conservent leur ordre initial après le tri.

4.2 Tri par insertion (Insertion Sort)

On construit progressivement une liste triée. Chaque nouvel élément est inséré à la bonne position.

Pour i = 2 à n
    clé = A[i]
    j = i - 1

    Tant que j > 0 et A[j] > clé
        A[j+1] = A[j]
        j = j - 1

    A[j+1] = clé

Complexité :

  • Meilleur cas (tableau trié) : O(n)
  • Pire cas (ordre inverse) : O(n²)
  • Stable et in-place
4.3 Tri par sélection (Selection Sort)

À chaque étape, on cherche le plus petit élément et on l'échange avec la première position non triée.

Pour i = 1 à n-1
    min = i

    Pour j = i+1 à n
        si A[j] < A[min]
            min = j

    échanger A[i] et A[min]

Nombre de comparaisons :

(n-1) + (n-2) + ... + 1 = n(n-1) / 2

Donc : T(n) = O(n²)

Propriétés : in-place, généralement non stable.

4.4 Tri à bulles (Bubble Sort)

On compare les éléments adjacents et on les échange si nécessaire. Les plus grands éléments remontent vers la fin du tableau.

Pour i = 1 à n-1
    Pour j = 1 à n-i
        si A[j] > A[j+1]
            échanger A[j] et A[j+1]

Complexité :

  • Pire cas : O(n²)
  • Meilleur cas (tableau trié, avec optimisation) : O(n)
  • Stable et in-place
4.5 Comparaison des algorithmes de tri
Algorithme Meilleur cas Pire cas Stable Mémoire
Insertion Sort O(n) O(n²) Oui In-place
Selection Sort O(n²) O(n²) Non In-place
Bubble Sort O(n) O(n²) Oui In-place

Ces tris simples ont généralement une complexité O(n²). Pour de grandes données, on utilise des algorithmes plus efficaces comme Merge Sort, Quick Sort ou Heap Sort qui atteignent O(n log n).

Section 05

Application pratique (contexte réel)

Dans le développement logiciel, l'analyse asymptotique permet d'anticiper les performances d'un programme avant même son exécution.

Par exemple, dans une application manipulant une grande base de données, un algorithme de recherche linéaire en O(n) peut devenir inefficace lorsque le nombre d'éléments devient très élevé. L'utilisation de structures de données optimisées comme les tables de hachage ou les arbres équilibrés permet de réduire la complexité à O(log n) ou O(1).

Dans le contexte du développement blockchain, cette optimisation est encore plus critique car chaque opération consomme des ressources calculatoires mesurées en gas.
