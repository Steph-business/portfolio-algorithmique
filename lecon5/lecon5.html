Leçon 5 : Tables de hachage

Section 01

Journal d'apprentissage réflexif

Cette leçon m'a permis de découvrir les tables de hachage, une structure de données qui permet de stocker et retrouver des informations très rapidement grâce à une clé. J'ai compris que contrairement aux tableaux ou aux listes où la recherche prend O(n), les tables de hachage atteignent O(1) en moyenne.

Le concept de fonction de hachage m'a intéressé : transformer une clé en indice de tableau est une idée élégante. Mon principal défi reste la compréhension des différentes méthodes de résolution de collisions, notamment la différence entre sondage linéaire, quadratique et double hachage.

Section 02

Synthèse des concepts clés

2.1 Principe des tables de hachage
Définition : Table de hachage

Une table de hachage est un tableau qui stocke des paires clé-valeur. On applique une fonction de hachage à la clé pour obtenir un indice dans le tableau.

Comparaison avec d'autres structures :

Structure Temps moyen
Tableau / Liste O(n)
Arbre binaire équilibré O(log n)
Table de hachage O(1)
2.2 Fonction de hachage

Une fonction de hachage transforme une clé en indice. Elle doit être :

  • Rapide à calculer
  • Déterministe : même clé → même résultat
  • Uniforme : répartir les clés uniformément
Méthode de division
h(k) = k mod m
Exemple : k = 100, m = 13 → h(100) = 100 mod 13 = 9
Méthode multiplicative
h(k) = ⌊ m (kA mod 1) ⌋

avec 0 < A < 1, souvent A ≈ (√5 − 1) / 2

Avantage : m n'a pas besoin d'être premier.

2.3 Collisions
Définition : Collision

Une collision se produit lorsque deux clés donnent le même indice. Il faut alors résoudre la collision.

2.4 Résolution des collisions
Chaînage séparé

Chaque case du tableau contient une liste chaînée. Si collision, on ajoute dans la liste.

Avantage : simple. Inconvénient : consomme plus de mémoire.

Adressage ouvert : Sondage linéaire

On teste les cases suivantes : i, i+1, i+2, i+3...

Problème : clustering primaire (regroupement d'éléments).

Adressage ouvert : Sondage quadratique

On teste : i, i+1², i+2², i+3² → i, i+1, i+4, i+9, i+16...

Réduit le clustering mais ne l'élimine pas.

Adressage ouvert : Double hachage

On utilise 2 fonctions de hachage h(k) et h'(k).

Positions testées : h(k), h(k)+h'(k), h(k)+2h'(k), h(k)+3h'(k)...

Avantage : réduit fortement les collisions.

2.5 Performance et facteur de charge
Facteur de charge
α = n / taille_table
Exemple : n = 75, table = 100 → α = 0.75

La valeur idéale est environ α ≈ 0.75.

Quand la table est trop pleine, on effectue un redimensionnement : on crée une nouvelle table plus grande et on rehache tous les éléments.

Section 03

Application pratique dans mon contexte

Les tables de hachage sont omniprésentes en développement logiciel :

  • Les dictionnaires en Python et les objets en JavaScript sont implémentés avec des tables de hachage.
  • Les bases de données utilisent des index de hachage pour accélérer les requêtes de recherche.
  • Les caches (mémoire cache, cache web) reposent sur des tables de hachage pour un accès rapide aux données fréquemment utilisées.
  • La blockchain utilise des fonctions de hachage cryptographiques pour garantir l'intégrité des transactions.
Section 04

Auto-évaluation et méta-cognition

Ce que je maîtrise bien :

✓ Principe des paires clé-valeur ✓ Méthode de division h(k) = k mod m ✓ Notion de collision et chaînage séparé

Ce que je dois améliorer :

↗ Sondage quadratique et double hachage ↗ Choix du facteur de charge optimal

Stratégie pour progresser :

  • Implémenter une table de hachage complète avec chaînage séparé
  • Comparer les performances des différentes méthodes de résolution de collisions
  • Étudier les fonctions de hachage utilisées en cryptographie
